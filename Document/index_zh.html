AI 辅助 UMG 工作流

通过明确的 MCP 工具创建 Unreal UMG 界面。

UMG MCP 让 Agent 读取真实 Widget Tree，修改布局和 Slot，应用 Blueprint 逻辑，编辑材质，创建 UMG 动画，并把 UI 资产作为结构化 JSON 做导出与回写。

安装

Fab 版

适合需要市场安装包和编辑器内工作流的项目。

  1. 从 Fab 为 Unreal Engine 5.8 安装 UMG MCP。
  2. 打开项目，启用 UmgMcp，然后重启编辑器。
  3. 在插件设置中选择 Google OAuth、Gemini API、智谱 AI、OpenAI 兼容端点或本地模式。
  4. 使用编辑器内对话面板，或连接外部 MCP 客户端。

源码版

适合协议开发、Issue 修复或手动集成。

cd D:\UE5Project\YourProject
mkdir Plugins
cd Plugins
git clone https://github.com/winyunq/UnrealMotionGraphicsMCP.git UmgMcp

然后用 UE 5.8 打开项目，让 Unreal Build Tool 编译插件。

连接 MCP 客户端

先打开 Unreal Editor。每个编辑器会发布独立的系统分配端点，Python MCP server 会发现并连接所选编辑器桥接。

{
  "mcpServers": {
    "UmgMcp": {
      "command": "uv",
      "args": [
        "run",
        "--directory",
        "D:\\UE5Project\\YourProject\\Plugins\\UmgMcp\\Resources\\Python",
        "UmgMcpServer.py"
      ]
    }
  }
}
必须使用绝对路径。
目录必须是 Resources/Python。
Windows 路径要转义。
JSON 中写成 D:\\Path。
先验证再做大任务。
优先运行 APITest 下的测试脚本。

排错

现象处理方式
ConnectionRefusedError 或 WinError 10061打开 Unreal Editor，启用插件，重启编辑器，然后重新连接 MCP 客户端。
系统找不到 uv安装 uv，或在 Resources/Python 下创建本地虚拟环境。
Agent 改错了 Widget先设置当前 UMG 目标，或传入完整资产路径，例如 /Game/UI/WBP_Login.WBP_Login。
