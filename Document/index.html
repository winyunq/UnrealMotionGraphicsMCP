Install, connect, and verify the Unreal UMG MCP bridge.

This manual keeps the practical workflow in one place: install the plugin, connect your MCP client, verify the editor bridge, and then use the protocol references for exact payloads.

Install

Source install

Clone the open-source plugin into your Unreal project and let Unreal Build Tool compile it.

cd D:\UE5Project\YourProject
mkdir Plugins
cd Plugins
git clone https://github.com/winyunq/UnrealMotionGraphicsMCP.git UmgMcp

Fab package

Use Fab only when you want the marketplace package. It is a distribution path for the same public UMG MCP project story, not a separate documentation site.

Optional Fab listing

Connect an MCP client

Start Unreal Editor first. The plugin listens on 127.0.0.1:55557, and the Python MCP server forwards tool calls to that editor bridge.

{
  "mcpServers": {
    "UmgMcp": {
      "command": "uv",
      "args": [
        "run",
        "--directory",
        "D:\\UE5Project\\YourProject\\Plugins\\UmgMcp\\Resources\\Python",
        "UmgMcpServer.py"
      ]
    }
  }
}

Verify

Static checks

cd D:\UE5Project\YourProject\Plugins\UmgMcp\Resources\Python
uv run python APITest\Umg_Widget_Protocol_Static_Check.py
uv run python APITest\Blueprint_MCP_Schema_Check.py
uv run python APITest\Material_Protocol_Static_Check.py
uv run python APITest\Animation_Protocol_Static_Check.py

Editor smoke test

uv run python APITest\UE5_Editor_Imitation.py

If this cannot connect, reopen Unreal Editor, confirm the plugin is enabled, and check whether another process owns port 55557.
