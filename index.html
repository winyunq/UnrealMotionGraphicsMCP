第 4 步 · 理解连接

两条入口，最终进入同一个 C++ 命令核心

“Fab 不需要 Python”只适用于 Fab 的编辑器内对话路线；Fab 也可以走外部 MCP 路线，而那条路线仍使用 Python stdio server。

开源版与 Fab 都可用

外部 MCP 路线

AI 客户端→ Python MCP stdio→ 动态本机端口→ UE C++ Bridge→ 资产
  • 公开的 89 工具都属于这条 MCP 工具面。
  • 多个 UE 实例通过发现记录和动态端口区分。
  • 写操作串行进入 Game Thread，避免并发修改资产。
仅 Fab 内置

编辑器内原生路线

Slate Chat→ Provider / Agent→ Tool Mode→ C++ Bridge→ 资产
  • 不启动 Python server，工具调用在插件内直接派发。
  • Tool Mode 按任务筛选工具，不等于默认暴露完整 89 工具。
  • Task、历史、审批、上下文压缩和本地模型属于这一层。
第 5 步 · 执行任务

任何可靠的编辑都遵循同一条闭环

不要从“生成 UI”直接跳到写入；先连接、定目标、读取真实状态，再做最小修改并验证。

  1. 01

    启动与连接

    打开 UE；外部客户端先 list_umg_mcp_servers，再 connect_umg_mcp。Fab 内置路线直接打开 UMG AI Assistant。

  2. 02

    选择目标

    调用 set_target_umg_asset / set_target_blueprint_asset，必要时再聚焦 Widget、函数或动画。

  3. 03

    读取真实状态

    读取 Widget Tree、属性、布局、BlueCode roundtrip、动画 overview 或材质图，而不是根据截图猜测。

  4. 04

    规划最小修改

    优先使用 union 写入；BlueCode 修改已有图时携带 base_revision/source_map，并可先 dry_run。

  5. 05

    显式处理删除

    删除 Widget、节点、变量、动画、轨道、关键帧或 HLSL 项时，使用对应 delete 工具并传 confirm_delete=true。

  6. 06

    编译与保存

    Blueprint 使用 bluecode_compile/compile_blueprint，HLSL 使用 hlsl_compile，其他资产使用 save_asset。

  7. 07

    回读验证

    重新读取刚修改的范围，检查结构、属性、连接、关键帧和编译诊断，再结束任务。

版本状态与已知边界

文档明确写出“现在能做什么，也写出还不能保证什么”

发布线

Fab 为 v1.0.13 / 内部版本 9；开源 MCP 核心为 v1.0.12 / 内部版本 8。Fab 构建覆盖 UE 5.6、5.7、5.8，Win64；本次使用全新的制品 URL 触发 Fab 重新扫描。

BlueCode 仍在过渡期

union、dry-run、revision、source map、右锚点和失败回滚已实现；任意嵌套 CFG、循环、Sequence/Switch、Latent 与不可规约图仍是后续工作。

工具面不是同一个概念

公开 89 工具、Fab 内置 Agent 的 Tool Mode、以及 C++ 原始 CommandType 是三个层次；本页不会再把它们混算。

安全写入

增量写入默认保留未提及内容；破坏性操作必须走专用删除工具并显式确认。修改后仍应回读和编译验证。
